2.10 RAD vs Waterfall

← Topic 2

RAD and the Classical Waterfall represent opposite ends of the SDLC spectrum: Waterfall prioritises planning and documentation; RAD prioritises speed and user feedback.

ParameterRADWaterfall
SpeedRAD is very fast because work is organised in time-boxed iterations with rapid delivery.Waterfall is slow because the customer receives the product only after the full cycle completes.
User involvementUsers are involved continuously through workshops and prototyping sessions.Users are involved mainly during requirements definition and final acceptance.
RequirementsRequirements are allowed to evolve during development as users learn more.Requirements are fixed upfront in the SRS before construction begins.
DocumentationRAD keeps documentation minimal and prefers working software as the main deliverable.Waterfall produces extensive documentation at each phase.
ToolsRAD relies heavily on CASE tools and code generation.Waterfall typically uses standard development tools.
Team sizeRAD works best with small, skilled teams working closely with users.Waterfall can support teams of any size, including very large teams.
FlexibilityRAD is highly flexible because scope is adjusted to fit each time-box.Waterfall has low flexibility because changes become costly after sign-off.
Best forRAD is best for small or medium applications, tight deadlines, and highly engaged users.Waterfall is best for fixed requirements, regulatory environments, and large documented projects.
Exam question — RAD vs Waterfall

Q: Compare RAD with the Traditional Waterfall SDLC. When would you choose each?

Model answer points:

  • Present a comparison using criteria such as speed, user involvement, requirements handling, documentation, tools, team size, and flexibility.
  • Explain when RAD is appropriate, for example when the schedule is tight, users are available, scope is moderate, and CASE tools exist.
  • Explain when Waterfall is appropriate, for example when requirements are stable, an audit trail is required, or the project is large and regulated.
