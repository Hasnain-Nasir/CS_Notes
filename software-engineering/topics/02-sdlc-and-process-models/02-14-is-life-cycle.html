2.14 Information System Life Cycle

← Topic 2

The Information System (IS) Life Cycle describes phases specific to developing information systems that collect, process, store, and distribute information for organisational decision-making. It extends generic SDLC with IS-specific concerns such as data migration, user training, and organisational change management.

IS-specific phases

  1. Initiation / feasibility. The organisation identifies the business need, defines scope, performs cost–benefit analysis, and obtains stakeholder approval.
  2. Requirements determination. Analysts study the information needs of users and departments that the system will support.
  3. Analysis. The team models current and proposed processes using tools such as data-flow and entity–relationship diagrams and performs gap analysis.
  4. Design. Architects define system architecture, database design, user interface, security, and integration interfaces.
  5. Implementation. Developers code the system, convert data, test the solution, and train users.
  6. Operation and maintenance. The organisation monitors performance, handles change requests, and maintains backups while the system is live.
  7. Retirement / disposal. When the system is replaced, data is migrated and the legacy system is decommissioned safely.

IS-specific concerns

  • Organisational impact. A new information system changes workflows, so change management and communication are critical.
  • Data quality. Poor source data leads to poor decisions, so data cleansing during migration is essential.
  • Security and compliance. The system must provide access control, audit trails, and compliance with relevant regulations.
  • User adoption. Non-technical staff need training, help-desk support, and clear documentation to use the system effectively.
Exam question — IS Life Cycle

Q: Describe the Information System Life Cycle. How does it differ from generic SDLC?

Model answer points:

  • Define the Information System Life Cycle as the set of phases used to build systems that manage organisational information and support decisions.
  • List the phases in order from initiation through requirements, analysis, design, implementation, operation, and retirement.
  • Explain IS-specific concerns such as organisational change, data migration, user training, compliance, and data quality.
  • Contrast with generic SDLC by noting that IS life cycles explicitly include business processes and people, not only software construction.
