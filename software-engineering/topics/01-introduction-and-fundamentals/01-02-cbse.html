1.2 Computer-based System Engineering

← Topic 1

Computer-based System Engineering (CBSE) is an engineering discipline concerned with designing and building complete systems in which software plays a central role but is not the whole picture. A computer-based system includes hardware, software, people, data, procedures, and documentation working together to achieve an organisational or user goal.

Software Engineering is a part of CBSE. When you engineer a hospital information system, an air-traffic control system, or a university portal, you must consider networks, servers, databases, user interfaces, staff training, operational procedures, and legal compliance — not just source code.

Dual role of software

Software plays two distinct roles in industry:

1. Software as a product

  • Software delivers computing potential across networks of hardware by running instructions on physical machines.
  • Software enables hardware to provide its expected functionality, because hardware alone cannot perform useful tasks without programs.
  • Software acts as an information transformer because it produces, manages, acquires, modifies, displays, or transmits information for users and organisations.

2. Software as a vehicle for delivering a product

  • Software provides the core functionality of many systems, such as a payroll system that calculates and records employee salaries.
  • Software controls other software, as an operating system manages and runs application programs on a computer.
  • Software helps build other software through tools such as compilers, IDEs, and testing frameworks that support the development process.

Example — University course portal

A course notes website is a computer-based system. Its software includes the HTML, CSS, JavaScript, and hosting platform that serve the pages. Hardware consists of servers and the devices students use to access the site. People include students who read the notes and the author who maintains them. Data is the lecture content stored and displayed on the site. Procedures describe how users navigate topics and follow the syllabus. Documentation maps each page to syllabus items. CBSE ensures all these elements are designed and integrated coherently.

Exam question — CBSE vs Software Engineering

Q: What is Computer-based System Engineering? How does it relate to Software Engineering?

Model answer points:

  • CBSE is the engineering of complete systems that combine hardware, software, people, data, procedures, and documentation.
  • Software Engineering is the disciplined development of the software component within such computer-based systems.
  • Software plays a dual role because it can be a product itself and also a vehicle or tool for delivering other products.
  • The syllabus topic "Introduction to Computer-based System Engineering" covers this broader systems view beyond software alone.
