1.8 Software Evolution

← Topic 1

Software evolution is the process of developing software and then continuously updating it over time — adding features, improving performance, fixing defects, or removing outdated functionality. The environment surrounding software keeps changing, so static software quickly becomes inadequate.

Why software must evolve

  • Changing requirements: Organisations adopt new workflows and business rules, so software must adapt to remain useful.
  • Environmental changes: New operating system versions, hardware, regulations, and security threats force software to be updated.
  • New features and performance: Users expect improvements over time, and competitors also push organisations to release updated versions.
  • Security: Outdated software becomes vulnerable to cyberattacks and must be patched or replaced.

Challenges of evolution

  • Increasing cost: As software grows larger, testing and debugging each change becomes more expensive.
  • Loss of understanding: When original developers leave, the remaining team may find the code hard to modify safely.
  • Security risks: Each update can introduce new vulnerabilities if changes are not reviewed and tested carefully.
  • Compatibility problems: Updates may break linked systems or third-party components that depend on the old version.
  • Pressure to ship quickly: Rushed updates often receive insufficient testing and result in poor quality releases.

Evolution of Software Engineering as a discipline

Software development evolved from an art (individual "hacking" and intuition) to a recognised engineering discipline:

  • 1960s–70s: Structured programming and flowcharts emerged, and the term "software engineering" was coined at a NATO conference.
  • 1980s: Object-oriented programming changed the way software systems were designed and organised.
  • 1990s: Agile methodologies emphasised flexibility, rapid delivery, and close customer feedback.
  • Today: Software Engineering uses established best practices, process models, metrics, and tools like other engineering disciplines.

Lehman's laws (exam context): large programs are always changed; complexity increases unless work is done to reduce it; evolution is a self-regulating process. Good SE practices (modularity, documentation, testing) make evolution cheaper and safer.

Exam question — Software Evolution

Q: What is software evolution? Why is it necessary and what challenges does it present?

Model answer points:

  • Software evolution is the ongoing change and updating of software after its initial delivery.
  • It is necessary because of new requirements, changing environments, demand for new features, and security concerns.
  • Its main challenges include rising cost, growing complexity, compatibility problems, and loss of knowledge about the system.
  • The Software Engineering discipline evolved from an art to a formal engineering field to manage this reality more effectively.
