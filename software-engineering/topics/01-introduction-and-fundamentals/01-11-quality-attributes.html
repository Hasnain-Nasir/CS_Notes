1.11 Software Quality Attributes

← Topic 1

Software quality attributes are measurable or assessable properties that describe how good a software product is. They guide design decisions and are used in acceptance criteria, reviews, and exams as distinct from general "fitness of purpose."

  • Correctness: The software performs its intended tasks and meets user requirements accurately.
  • Usability: The software is easy to learn, operate, and navigate for different classes of users.
  • Reliability: The software produces consistent and accurate results with stable performance and fewer failures over time.
  • Efficiency: The software makes optimal use of CPU, memory, disk, network, and other system resources.
  • Maintainability: The software is easy to correct, adapt, and enhance when requirements change.
  • Portability: The software can run on different platforms or environments with minimal modification.
  • Scalability: The software can handle increased workload or more users without a serious drop in performance.
  • Security: The software protects data and functions against unauthorised access, breaches, and other threats.
  • Modularity: The software is organised into separate, manageable components that can be updated independently.
  • Reusability: Software components can be used again in other applications to save development time and cost.
  • Testability: The software makes it easy to define test criteria and evaluate the system against them.

Two further attributes are adaptability, which means the software can satisfy changing constraints, and interoperability, which means it can work cooperatively with other systems.

Exam question — Quality Attributes

Q: Define software quality attributes. Explain any five with examples.

Model answer points:

  • The answer should select any five attributes, such as correctness, reliability, maintainability, portability, and security.
  • Each selected attribute should be defined clearly and supported with a concrete example, such as a banking app that must be secure and correct.
  • Quality attributes should be distinguished from the broader concept of software quality discussed in §1.12.
