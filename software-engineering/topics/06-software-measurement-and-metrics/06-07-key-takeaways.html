Key takeaways — exam revision

← Topic 6
  • Measurement vs metric: Measurement = raw attribute value; metric = quantitative indicator for decision-making. Five activities: formulation, collection, analysis, interpretation, feedback.
  • Metric types: Product (size, complexity — LOC, FP, Halstead, cyclomatic); Process (lead time, defect density, compliance); Project (cost, schedule, staffing, productivity).
  • Good metric properties: Quantitative, understandable, repeatable, economical, applicable early, language independent.
  • Direct vs indirect: Direct = measure attribute itself (LOC, FP count); indirect = estimate via related parameters (productivity-based effort).
  • People metrics: Productivity, eNPS, team collaboration, attrition, absenteeism, total workforce cost, quality of work. Focus on workforce performance and satisfaction.
  • Process metrics: Static (process structure), dynamic (runtime performance), evolution (changes over time). Top metrics: lead time, cycle time, throughput, WIP, defect density, efficiency, compliance. Link to ISO 9000 and COCOMO.
  • Exam classification: Size & complexity = Product metrics (UGC-NET). Language-dependent attribute should NOT be in good metrics.
  • Halstead (1977): Program = operators + operands. n1, n2 = distinct; N1, N2 = total. N=N1+N2; n=n1+n2; V=N·log₂(n); D=(n1/2)·(N2/n2); E=V·D; T=E/(18×60). Errors≈V/3000. Requires complete code; size-oriented white-box metric.
  • Function Points (Albrecht 1979): User-oriented black-box size metric; language independent. Five components: EI, EO, EQ, ILF, EIF. Weights: EI(3/4/6), EO(4/5/7), EQ(3/4/6), ILF(7/10/15), EIF(5/7/10). UFP=Σ(count×weight); CAF=0.65+0.01×Σ(fi); FP=UFP×CAF. 14 adjustment factors (0–5 each); CAF range 0.65–1.35.
  • LOC: Non-comment, non-blank lines; KLOC=thousands. White-box/direct metric; language dependent. Effort=Size/Productivity; Cost=Effort×Pay. Used in COCOMO (DE=a·KLOC^b). Correlates roughly with cost, duration, defects.
  • LOC vs FP: LOC=design-oriented, language-dependent, after coding; FP=specification-based, user-oriented, language-independent, early estimation. Both measure software size for effort/cost models.
  • Cyclomatic complexity (McCabe): Control-flow complexity from CFG. V(G)=E−N+2; V(G)=R+1; V(G)=D+1. Minimum basis path test cases = V(G). No control flow → V(G)=1; one if → V(G)=2. Low=simpler; high=refactor. Control complexity only — not data complexity.
  • Size metric families: Size-oriented (LOC, Halstead) = white-box; Function-oriented (FP) = black-box; Complexity (cyclomatic) = structural/control-flow.
  • Links to SPM: Size estimation §3.5 (direct KLOC vs indirect FP); COCOMO §3.6 (KLOC or FP input); decomposition estimation uses both approaches.
