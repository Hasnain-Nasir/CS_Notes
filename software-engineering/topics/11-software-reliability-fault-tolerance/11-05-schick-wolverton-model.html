11.5 Schick-Wolverton Model

← Topic 11

The Schick-Wolverton (SW) model (1978) is a modification of the §11.4 Jelinski-Moranda Model. It retains JM's assumption of a fixed initial fault count N and perfect debugging, but introduces a time-dependent failure rate that increases within each failure interval.

Motivation

  • In the JM model, the failure intensity λ is constant within each interval between failures — it only changes when a fault is removed.
  • In practice, the longer a fault remains undetected, the more likely it is to cause a failure as more execution paths are exercised.
  • The SW model captures this by making λ proportional to the time elapsed since the last debugging event.
  • SW is more sophisticated than JM and generally provides better fits to real failure data.

Assumptions

  1. The software contains exactly N initial faults at the start of testing.
  2. Each remaining fault has hazard rate φ, but the overall failure intensity increases linearly with time since the last fault was removed.
  3. Failures occur independently.
  4. Perfect debugging: faults are permanently removed upon detection; no new faults are introduced.

Failure intensity

During the ith failure interval, where ti is the time elapsed since the (i − 1)th failure was debugged:

λ(ti) = φ [N − (i − 1)] ti

  • At ti = 0 (immediately after debugging), λ = 0 — no failure can occur instantly after a fix.
  • As ti increases, λ grows linearly — reflecting increased exposure of remaining faults.
  • When a new failure occurs and a fault is removed, λ resets to 0 and the coefficient drops to φ[N − i].

Reliability function

The probability of failure-free operation during the ith interval is:

R(ti) = e−φ [N − (i − 1)] ti² / 2

  • The exponent contains ti² rather than ti because λ(ti) is linear in ti — integrating λ gives a ti²/2 term.
  • Reliability decreases more rapidly early in the interval (when λ is small) and more slowly later — a different shape from JM's exponential decay.

Parameter estimation

  • Both N and φ are estimated from Type-I failure data using Maximum Likelihood Estimation (MLE).
  • Estimating N is particularly important — it represents the total inherent fault content and directly affects reliability predictions.
  • The MLE for N involves maximising the likelihood function over integer values N ≥ n (where n is the number of observed failures).
  • φ is estimated simultaneously; the log-likelihood for the SW model incorporates the ti² term from the reliability function.
Exam question — Schick-Wolverton Model

Q: How does the SW model modify JM? Write the failure intensity and reliability formulas. Why is SW more sophisticated?

Model answer points:

  • SW (1978) = modification of JM; same N and perfect debugging assumptions.
  • Key change: failure rate increases with time since last debugging — λ(ti) = φ[N − (i − 1)]ti.
  • R(ti) = e−φ[N−(i−1)]ti²/2.
  • λ resets to 0 after each fault removal; grows linearly within interval.
  • More sophisticated: captures time-dependent exposure; better fit than JM.
  • Parameters N, φ estimated via MLE from Type-I data.
