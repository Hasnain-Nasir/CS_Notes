11.8 Mills' Error Seeding Model

← Topic 11

Mills' Error Seeding Model (1980s) is a technique for estimating the total number of inherent faults in software by injecting a known number of artificial errors and measuring how many are detected during testing. It is covered in the SRGM overview at §11.3 Reliability Growth Models as a fault-content estimation approach rather than a time-based growth model.

Principle

  • The tester injects a known number of seeded (artificial) errors into the program alongside the inherent (natural) errors already present.
  • Testing is then performed without distinguishing seeded from inherent errors — the tester finds some of both types.
  • The ratio of detected seeded errors to total seeded errors is assumed to equal the ratio of detected inherent errors to total inherent errors — this is the capture-recapture principle.
  • From this ratio, the total number of inherent faults can be estimated.

Variables

SymbolMeaning
SNumber of seeded (known) errors injected into the program
sNumber of seeded errors detected during testing
N₀Estimated number of inherent (natural) errors in the program
n₁Total number of errors detected during testing (seeded + inherent)
kNumber of inherent errors detected during testing (n₁ − s)
rTotal number of errors in the combined population (S + N₀)
N̂Estimated total inherent error count (rounded up)

Hypergeometric distribution approach

The error seeding problem is modelled using the hypergeometric distribution, which describes the probability of drawing a certain number of successes from a finite population without replacement.

  • Population size: r = S + N₀ (seeded + inherent errors).
  • Sample size: n₁ (total errors detected).
  • Successes in population: S (seeded errors).
  • Successes in sample: s (seeded errors detected).

The estimate for inherent errors is:

N₀ = n₁(r − k) / k − 1

The estimated total inherent error count (rounded up to the next integer) is:

N̂ = ⌈N₀⌉ + 1

  • Here r = S + N₀ and k = n₁ − s (inherent errors found).
  • The formula assumes seeded and inherent errors have equal probability of being detected — a critical assumption.

Alternative: two-programmer method

A simpler estimation formula used in the two-programmer variant is:

N̂ = n₁r / k

  • In this variant, one programmer seeds S known errors, and a second programmer (tester) finds n₁ total errors without knowing which are seeded.
  • r = S + N₀ as before, and k = n₁ − s.
  • This method is easier to apply but makes the same equal-detection-probability assumption.

Steps of error seeding

  1. Introduce errors: A known number S of artificial errors is seeded into the program by an experienced programmer who records their locations.
  2. Test: The program is tested by an independent tester who does not know which errors are seeded and which are inherent.
  3. Measure: The total errors found n₁ and the seeded errors found s are counted; inherent errors found k = n₁ − s.
  4. Estimate and improve: The total inherent error count N̂ is computed; if N̂ is too high, more testing or debugging is performed and the process may be repeated.

Error seeding estimation — pseudocode

error_seeding.pseudo 
// Step 1: Seed known errors into the program
S = 20                    // number of seeded errors injected
seed_locations = INJECT_ERRORS(program, S)

// Step 2: Independent tester runs tests (blind to seed locations)
test_results = RUN_TESTS(program)
n1 = COUNT(test_results.errors_found)   // total errors found

// Step 3: Identify how many seeded errors were detected
s = COUNT(test_results.errors_found WHERE error IN seed_locations)
k = n1 - s                // inherent errors detected

// Step 4: Estimate total inherent errors (hypergeometric method)
// N0 = n1 * (r - k) / k - 1,  where r = S + N0
// Solve iteratively or use two-programmer shortcut:
N_hat = CEIL(n1 * (S + N0_estimate) / k)
// Simplified two-programmer formula:
N_hat_simple = n1 * (S + N0_estimate) / k

// Step 5: Decision
IF N_hat > acceptable_threshold THEN
  RECOMMEND more testing or debugging
ELSE
  RECOMMEND sufficient fault removal for release
END IF

Drawbacks and limitations

  • Cost: Seeding errors requires an experienced programmer's time to inject realistic faults, and a separate tester must be available — doubling personnel requirements.
  • Equal detection assumption: Seeded errors must be equally likely to be detected as inherent errors — if seeded errors are too obvious or too subtle compared to real faults, the estimate is biased.
  • Representativeness: Artificially seeded errors may not match the type, severity, or location distribution of real inherent errors.
  • Tester awareness: If the tester suspects seeding is in use, they may change their testing strategy, invalidating the statistical assumptions.
  • Small sample bias: When few errors are detected (small k), the estimate N̂ becomes highly unstable and unreliable.
Exam question — Mills' Error Seeding Model

Q: Explain Mills' error seeding technique. Write the estimation formulas. List the steps and drawbacks.

Model answer points:

  • Inject S known errors; test blindly; use capture-recapture to estimate inherent fault count.
  • Hypergeometric: N₀ = n₁(r − k)/k − 1; N̂ = ⌈N₀⌉ + 1.
  • Two-programmer method: N̂ = n₁r/k.
  • Steps: introduce errors → test → measure (n₁, s, k) → estimate and improve.
  • Drawbacks: cost (two personnel); seeded errors must match detection likelihood of inherent errors; representativeness; small sample bias.
  • Variables: S (seeded), s (seeded found), n₁ (total found), k = n₁ − s (inherent found), N̂ (estimated inherent total).
