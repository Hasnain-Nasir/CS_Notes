11.6 Goel-Okumoto Model

← Topic 11

The Goel-Okumoto (GO) model (1979) is an exponential Non-Homogeneous Poisson Process (NHPP) model. Unlike the step-function §11.4 Jelinski-Moranda Model, GO produces a smooth, continuously decreasing failure intensity curve.

Model type

  • GO is an NHPP model — failures occur according to a Poisson process whose rate λ(t) changes over time.
  • It is one of the most widely used SRGMs because of its simplicity, two-parameter form, and good empirical fit.
  • The mean value function m(t) has an exponential saturation shape — failures are detected rapidly at first, then more slowly as remaining faults decrease.

Assumptions

  1. Faults in the software are independent of each other.
  2. The failure rate at any time is proportional to the number of remaining (undetected) faults.
  3. When a fault causes a failure, it is removed from the program before the next test session — faults are not rediscovered.
  4. Perfect debugging: every detected fault is permanently and correctly fixed; no new faults are introduced during debugging.

Differential equation

Let m(t) be the expected cumulative number of faults detected by time t, and a be the total initial fault content. The rate of fault detection is proportional to remaining faults:

dm/dt = b [a − m(t)]

  • The left side is the rate at which new faults are detected per unit time.
  • The right side says detection rate = b × (faults not yet found) = b[a − m(t)].
  • As m(t) approaches a, the detection rate approaches zero — all faults have been found.

Mean value function

Solving the differential equation with m(0) = 0 gives:

m(t) = a (1 − e−bt)

  • At t = 0, m(0) = 0 — no faults detected yet.
  • As t → ∞, m(t) → a — all a faults are eventually detected.
  • The curve is S-shaped in cumulative failures vs time — rapid growth initially, then saturation.

Parameters

ParameterSymbolMeaning
Total fault contentaTotal number of faults initially present in the software
Fault detection ratebRate at which remaining faults are detected per unit time

Failure intensity and reliability

  • Failure intensity: λ(t) = dm/dt = ab e−bt — decreases exponentially over time.
  • Reliability (probability of zero failures in [0, t]): R(t) = e−m(t) = e−a(1−e−bt).
  • Reliability increases as testing continues because m(t) grows (more faults removed) and λ(t) shrinks.

Parameter estimation (MLE)

  • Type-I data (time-between-failure): MLE maximises the likelihood using exact failure times t₁, t₂, …, tn. The log-likelihood involves terms ln λ(ti) and m(tn).
  • Type-II data (failure-count): MLE uses the number of failures ni observed in each time interval [ti−1, ti]. The likelihood is based on the Poisson probability of observing ni failures given m(ti) − m(ti−1).
  • Both a and b are estimated simultaneously; iterative numerical methods (e.g. Newton-Raphson) are typically used.
  • Initial estimates can be obtained from a plot of cumulative failures vs time — the saturation level estimates a, and the slope estimates b.

Exam question — Goel-Okumoto Model

Q: State the GO model assumptions. Write the differential equation, mean value function, and reliability formula. How are parameters estimated?

Model answer points:

  • GO (1979): exponential NHPP; smooth decreasing failure intensity.
  • Assumptions: independent faults; failure rate ∝ remaining faults; faults removed before retest; perfect debugging.
  • dm/dt = b[a − m(t)]; m(t) = a(1 − e−bt); λ(t) = ab e−bt.
  • R(t) = e−a(1−e−bt).
  • Parameters: a (total faults), b (detection rate) — MLE for Type-I and Type-II data.
  • Contrast with JM: GO has smooth curve, not step function; supports both data types.
