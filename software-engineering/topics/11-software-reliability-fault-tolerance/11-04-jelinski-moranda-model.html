11.4 Jelinski-Moranda Model

← Topic 11

The Jelinski-Moranda (JM) model (1972) is the first and one of the best-known black-box software reliability growth models. It belongs to the binomial type of SRGMs and is covered in the overview at §11.3 Reliability Growth Models.

Model type

  • The JM model is a binomial-type model — it assumes a fixed initial number of faults N, and each fault has an equal constant probability of causing the next failure.
  • It was among the earliest attempts to apply reliability growth theory (originally developed for hardware) to software testing data.
  • The failure intensity drops in discrete steps each time a fault is detected and removed — producing a step-function reliability growth curve.

Assumptions

  1. The software contains exactly N initial faults at the start of testing.
  2. Each remaining fault has the same constant hazard rate φ (phi) — the probability per unit time that a specific fault causes a failure.
  3. Failures occur independently — the failure caused by one fault does not affect the failure times of other faults.
  4. Perfect debugging: when a failure occurs, the fault that caused it is immediately and permanently removed — no new faults are introduced.

Failure intensity

After (i − 1) faults have been removed, (N − (i − 1)) faults remain. The failure intensity at the ith failure interval is:

λ(ti) = φ [N − (i − 1)]

  • At the start (i = 1), λ = φN — the highest failure rate when all N faults are present.
  • After each fault is removed, λ drops by φ — a step decrease.
  • When all N faults are removed, λ = 0 — the software is fault-free under this model's assumptions.

Reliability function

The probability of failure-free operation during the ith interval of duration ti (time since the (i − 1)th failure) is:

R(ti) = e−φ [N − (i − 1)] ti

  • Reliability increases as i increases (fewer remaining faults) and as ti increases (more time without failure in the current interval).
  • The reliability curve is exponential within each interval but steps up at each fault removal.

Parameters

ParameterSymbolMeaning
Initial fault countNTotal number of faults present at the start of testing
Fault hazard rateφConstant failure rate per individual fault per unit time
  • Both N and φ are estimated from Type-I (time-between-failure) data using Maximum Likelihood Estimation (MLE).
  • N must be an integer ≥ the number of failures observed; φ must be positive.

Optimistic predictions

  • The JM model tends to produce optimistic reliability predictions because it assumes perfect debugging and a fixed, finite fault count.
  • In practice, debugging may introduce new faults and the total fault count is not known in advance — the model overestimates how quickly reliability will improve.
  • Despite this limitation, JM remains important as the foundational SRGM and as a basis for later models such as the §11.5 Schick-Wolverton Model.
Exam question — Jelinski-Moranda Model

Q: State the JM model assumptions. Write the failure intensity and reliability formulas. Why are JM predictions optimistic?

Model answer points:

  • JM (1972): first black-box SRGM; binomial type; step-function failure intensity.
  • Assumptions: N initial faults; constant hazard φ per fault; independent failures; perfect debugging.
  • λ(ti) = φ[N − (i − 1)]; R(ti) = e−φ[N−(i−1)]ti.
  • Parameters: N, φ — estimated via MLE from Type-I data.
  • Optimistic because perfect debugging and fixed N overestimate reliability growth; constant φ unrealistic.
  • Modified by Schick-Wolverton (1978) — see §11.5.
