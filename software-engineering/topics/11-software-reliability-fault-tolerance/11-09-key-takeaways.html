Key takeaways — exam revision

← Topic 11
  • 11.1 Software Reliability: Probability of failure-free operation for fixed time in fixed environment; human-design dependent; no physical wear; lifecycle: many initial faults → testing → useful life → upgrades add new faults.
  • Reliability attributes: ROCOF (failure frequency), MTTF (avg time between failures — execution time only), MTTR (repair time), MTBF = MTTF + MTTR, POFOD (failure on demand), availability, fault tolerance, recoverability, redundancy.
  • Hardware vs software: Hardware = bathtub curve, wear, time-dependent; Software = no bathtub, no wear, redundancy can improve reliability, no preventive maintenance equivalent.
  • High reliability strategies: Error avoidance (experienced devs, tools) → error detection (testing) → error removal (debugging) → fault tolerance (NVP, RB, rollback).
  • 11.2 Fault Tolerance: Continue correct service despite faults; fault-removal (forward/backward recovery) vs fault-masking (NVP voting); design diversity.
  • Basic techniques: Redundancy, checkpointing, error detection/correction, load balancing, isolation, replication, dynamic reconfiguration.
  • Recovery Block (RB): Sequential — primary P, acceptance test T, alternates Q1..Qn; `Ensure T By P Else by Q1...`; rollback state on failure; better for non-real-time.
  • N-Version Programming (NVP): N independent versions run in parallel; voter selects result; no re-execution delay; controversial (common-mode failures).
  • Checkpoint/Rollback: Save system state periodically; restore on failure — backward error recovery.
  • 11.3 SRGMs: Mathematical models predicting reliability improvement during testing; black-box; guide testing and resource allocation; Type-I (failure-count) vs Type-II (time-between-failures) data.
  • 11.4 Jelinski-Moranda (1972): Binomial type; N initial faults; perfect debugging; λ(ti) = φ[N-(i-1)]; R(ti) = e^(-φ[N-(i-1)]ti); tends to be optimistic.
  • 11.5 Schick-Wolverton (1978): JM modification; failure rate increases with time since last failure: λ(ti) = φ[N-(i-1)]ti; R(ti) = e^(-φ[N-(i-1)]ti²/2).
  • 11.6 Goel-Okumoto (1979): Exponential NHPP; m(t) = a(1-e^(-bt)); a = total faults, b = detection rate; perfect debugging; widely used.
  • 11.7 PNZ Model: NHPP with imperfect debugging; S-shaped detection rate; faults introduced during debugging (rate α); a(t)=a(1+αt); evaluates fault tolerance benefit.
  • 11.8 Mills Error Seeding: Inject n₁ known errors; find k seeded in r removed; estimate N̂ = [n₁(r-k)/k - 1]+1; alternative: two programmers N̂ = n₁r/k; hypergeometric distribution.
  • Model comparison: JM/SW = time-between-failure; GO/PNZ = NHPP mean value; Mills = fault count estimation (not growth curve).
