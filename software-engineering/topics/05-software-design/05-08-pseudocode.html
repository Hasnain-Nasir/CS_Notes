5.8 Pseudo Code

← Topic 5

Pseudocode is a simplified, informal representation of an algorithm or program that uses a mix of natural language and programming constructs. It is used during the design phase to illustrate the high-level structure and logic of an algorithm without the syntactic details of a specific programming language.

Characteristics of pseudocode

  • Pseudocode is language-independent — it is not tied to any particular programming language syntax.
  • It uses familiar programming constructs (IF, WHILE, FOR, assignments) combined with plain English descriptions.
  • It focuses on the logic and steps of an algorithm, not on declaration syntax or memory management.
  • It is more structured than plain prose but less rigid than actual code.

Purpose of pseudocode in software design

  • Pseudocode helps developers plan and discuss algorithms before writing actual code.
  • It makes it easier to understand and translate the design into code later in the development process.
  • During detailed design, each module's algorithm is expressed in pseudocode before coding begins.
  • Pseudocode, along with structure charts and flowcharts, becomes part of the project documentation handed to programmers and testers.

Pseudocode vs algorithm vs code

AspectAlgorithmPseudocodeActual code
Formality Step-by-step procedure in natural language or mathematical notation. Structured mix of natural language and programming-like constructs. Strict syntax of a specific programming language.
When used Problem-solving, mathematical procedures. Design phase — planning module logic. Implementation (coding) phase.
Executability Not directly executable. Not directly executable. Directly executable by a compiler/interpreter.

Example — finding the maximum of three numbers

BEGIN
  INPUT num1, num2, num3
  SET max = num1
  IF num2 > max THEN
    SET max = num2
  END IF
  IF num3 > max THEN
    SET max = num3
  END IF
  OUTPUT max
END
  • This pseudocode can be directly translated into C, Java, Python, or any language without changing the logic.
  • Indentation and keywords (BEGIN, IF, SET, OUTPUT) make the control flow clear without language-specific syntax.

Example — calculating average marks

BEGIN
  INPUT total_students
  SET sum = 0
  SET count = 0
  WHILE count < total_students DO
    INPUT mark
    SET sum = sum + mark
    SET count = count + 1
  END WHILE
  SET average = sum / total_students
  OUTPUT average
END

Guidelines for writing good pseudocode

  • Use clear, descriptive variable names that reflect their purpose.
  • Use standard control structures (sequence, selection, iteration) consistently.
  • One logical step per line; use indentation to show nesting.
  • Avoid language-specific features (pointers, specific library calls) unless necessary.
  • Keep each pseudocode block focused on one module or one algorithm.
Exam question — Pseudocode

Q: What is pseudocode? Why is it used in the design phase? Write pseudocode for a simple task.

Model answer points:

  • Define: informal algorithm representation using natural language + programming constructs.
  • Purpose: plan algorithms, discuss logic, bridge design to code, part of documentation.
  • Properties: language-independent, not executable, structured.
  • Write small example (e.g., find max of three numbers or calculate average).
